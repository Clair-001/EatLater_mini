# 需求文档

## 介绍

EatLater 是一款专注于即时干预冲动性饮食行为的微信小程序。通过负面视觉刺激和暂停语义，帮助用户在产生强烈进食欲望但尚未完成下单或进食前的关键时刻，延迟或打断冲动决策。

## 术语表

- **系统**: EatLater 微信小程序
- **用户**: 有"想控制但偶尔失控"饮食行为的人群
- **食物输入**: 用户手动输入或选择想吃的食物名称的功能
- **反差视觉**: 降低食欲的食物图片展示功能
- **暂停引导**: 鼓励用户延迟决策的文案提示
- **快捷按钮**: 预设的常见食物选择按钮
- **图片库**: 预置的反差视觉图片集合

## 需求

### 需求 1: 食物输入功能

**用户故事:** 作为一个想要控制冲动饮食的用户，我希望能够快速输入我想吃的食物，这样系统就能为我提供相应的干预。

#### 验收标准

1. 当用户打开应用时，系统应当显示一个文本输入框用于输入食物名称
2. 当用户点击输入框时，系统应当激活键盘并允许文本输入
3. 系统应当提供3-5个快捷按钮（披萨、炸鸡、奶茶、蛋糕、其他常见食物）
4. 当用户点击快捷按钮时，系统应当自动填充对应的食物名称到输入框
5. 当用户输入或选择食物后，系统应当验证输入不为空并继续到下一步

### 需求 2: 反差视觉展示功能

**用户故事:** 作为一个想要控制冲动饮食的用户，我希望看到能够降低我食欲的食物图片，这样可以帮助我重新考虑我的饮食决策。

#### 验收标准

1. 当用户确认食物输入后，系统应当从预置图片库中选择对应的反差视觉图片
2. 系统应当展示明显降低食欲的对应食物图片，具有变质感、冷色调、违和感特征
3. 当没有找到对应食物的图片时，系统应当显示通用的反差视觉图片
4. 系统应当确保所有图片符合合规约束：不包含血腥、呕吐、人体病变内容
5. 系统应当确保图片不包含医疗、疾病、身体暗示内容

### 需求 3: 暂停引导功能

**用户故事:** 作为一个想要控制冲动饮食的用户，我希望收到温和的延迟提示，这样可以帮助我暂停冲动决策而不是感到被禁止。

#### 验收标准

1. 当反差视觉图片显示时，系统应当同时显示暂停引导文案
2. 系统应当从预设文案中选择一条显示，例如"先缓一缓，现在不急，晚点再决定"
3. 文案应当强调"延迟"概念而非"禁止"概念
4. 系统应当确保文案语调温和、支持性，避免批判性语言
5. 文案应当在屏幕上清晰可读，与图片形成有效的视觉组合

### 需求 4: 自然退出功能

**用户故事:** 作为一个想要控制冲动饮食的用户，我希望能够自然地退出应用，而不被强制进行其他操作，这样我可以自主决定下一步行动。

#### 验收标准

1. 当暂停引导显示后，系统应当提供明显的退出或关闭选项
2. 当用户选择退出时，系统应当直接关闭应用，不显示额外的引导或推荐
3. 系统应当不提供替代饮食推荐功能
4. 系统应当不强制用户进行后续操作或记录
5. 当用户退出后，系统应当清除当前会话数据，为下次使用做准备

### 需求 5: 应用启动和导航

**用户故事:** 作为一个在冲动饮食时刻的用户，我希望应用能够快速启动并直接进入核心功能，这样可以在关键的1-3分钟内有效干预。

#### 验收标准

1. 当用户启动应用时，系统应当在3秒内加载完成并显示食物输入界面
2. 系统应当直接显示核心功能，无需额外的登录或设置步骤
3. 当应用在后台运行时，系统应当能够快速恢复到主界面
4. 系统应当在微信小程序环境中稳定运行，支持主流微信版本
5. 当网络连接不可用时，系统应当仍能正常工作（使用本地图片库）
